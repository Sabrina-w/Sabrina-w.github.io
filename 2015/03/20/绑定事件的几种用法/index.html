绑定事件的方法

js原生方法

传统方法

例如

element.onclick = function(e){
    //...
}

优点

  • 简单和稳定，可以确保它在不同的浏览器中运行一致
  • 处理事件时，this指向当前元素

缺点

  • 一个元素只能绑定一个事件处理函数，新绑定的事件处理函数会覆盖旧的事件函数
  • 参数对象（e）IE不支持
  • 传统方法只在事件冒泡中运行

W3C方式

例如

element.addEventListerner('click',function(){
    //...
},false)

优点

  • 该方法同时支持事件的捕获和冒泡阶段。事件阶段取决于addEventListener函数的最后一个参数，为false时，为冒泡阶段，为true时，为捕获阶段
  • 可以为同一个元素绑定多个事件，不会被覆盖
  • 可以使用处理函数的第一个参数来捕获事件对象

缺点

  • IE不支持该函数，必须使用IE的attachEvent函数来代替

IE方式

例如

element.attachEvent('onclick',function(){
    //...
})

优点

  • 可以为同一个元素绑定多个事件，不会被覆盖

缺点

  • 该方法只支持事件的冒泡阶段
  • 事件处理函数中的this指向window对象，而不是当前元素
  • IE不支持处理函数中参数e捕获事件对象，事件对象仅存在window.event参数中
  • 事件名必须使用on+type方式
  • 仅限IE使用该方法，其他浏览器使用addEventListener().

jq方法

bind()

作用

bind() 方法为被选元素添加一个或多个事件处理程序，并规定事件发生时运行的函数。

用法1(单执行函数)

$(selector).bind(event,data,function)

参数

-> event:必需。规定添加到元素的一个或多个事件。由空格分隔多个事件。必须是有效的事件。
-> data:可选。规定传递到函数的额外数据。
-> functon: 必需。规定当事件发生时运行的函数。

用法2(多执行函数)

$(selector).bind({event1:function, event2:function, ...})

参数

event，function:都必须，用法同上。

优点

  • 可以向匹配的元素绑定一个或者多个事件处理器。

缺点

  • bind()函数只能针对已经存在的元素进行事件的设置，不可对动态添加的元素进行事件设置，即不可绑定委托事件。

live()

作用

live() 向当前或未来的匹配元素添加一个或多个事件处理器。

语法

-> 用法一（单执行函数）：$(selector).live(event,data,function);
-> 用法二（多执行函数）：$(selector).live({event1:function, event2:function, ...})

参数

 -> event:必需。规定添加到元素的一个或多个事件。由空格分隔多个事件。必须是有效的事件。
-> data:可选。规定传递到函数的额外数据。
-> functon: 必需。规定当事件发生时运行的函数。

优点

  • 可以绑定委托事件
  • 由于事件是绑定在document上的，可以在document ready之前就可以绑定那些需要的件。

缺点

  • 一个元素采用live方法进行事件的绑定的时候，如果想阻止事件的传递或冒泡，就要在函数中return false,仅仅调用stopPropagation()是无法实现阻止事件的传递或者冒泡的。
  • 由于事件是绑定在document上的，触发事件时需要从document一层一层的传递，速度很慢，效能低。
  • 从jq1.7引入on（）函数后，就不推荐使用了live（）函数了，在jq1.9之后取消了live（）函数。

delegate()

作用

为指定的元素（属于被选元素的子元素）添加一个或多个事件处理程序，并规定当这些事件发生时运行的函数。使用 delegate() 方法的事件处理程序适用于当前或未来的元素。

用法

用法一：$(selector).delegate(childSelector,event,data,function);
用法二：$(selector).delegate(childselector,{event1:function, event2:function, ...})　

参数

->childSelector： 必需。规定要附加事件处理程序的一个或多个子元素。
-> event: 必需，规定附加到元素的一个或多个事件。由空格分隔多个事件值。必须是有效的事件。
-> data: 可选；需要传递的参数.
-> function:必需；当绑定事件发生时，需要执行的函数

优点

  • 用delegate绑定委托事件时，事件时绑定在给定的祖先元素上的不，而不是document上，解决了live（）函数性能低的缺点。

缺点

  • 需要查找哪个元素上发生了那个事件，尽管比document少很多了，不过，还是得浪费时间来查找。
  • delegate函数只能绑定委托事件，不可绑定简单事件。

on

作用

为指定的元素,添加一个或多个事件处理程序，并规定当这些事件发生时运行的函数。使用 on() 方法的事件处理程序适用于当前或未来的元素

用法

用法一：$(selector).on(event,childselector,data,function)
用法二：$(selector).on({event1:function, event2:function, ...},childselector);

参数

->childSelector： 必需。规定要附加事件处理程序的一个或多个子元素。
-> event: 必需，规定附加到元素的一个或多个事件。由空格分隔多个事件值。必须是有效的事件。
-> data: 可选；需要传递的参数.
-> function:必需；当绑定事件发生时，需要执行的函数

优点

  • 提供了一种统一绑定事件的方法,可以用.on()来代替上述的3种方法。（其实.bind(), .live(), .delegate()都是通过.on()来实现的）

one（）

作用

作用以及用法与on（）一致，只是one（）函数在执行函数执行一次后自动移除元素的绑定事件，事件只执行一次。
